Перед началом любой задачи обязательно:

Открой файлы PROJECT_DOC.md и Changelog.md (или их аналоги).

Проанализируй и составь краткое summary по ключевым правилам:

Принцип DRY — избегай дублирования.

Общение, код и комментарии — только на русском языке.

Оформление изменений — всегда через Changelog.md по заданному шаблону.

Перед каждым действием изучи историю изменений.

При изменениях — синхронизируй с актуальным Changelog и делай запись.

Для каждой ключевой функции — unit-тесты; тесты всегда обновляются.

NO FALLBACK / NO MOCKING POLICY — никаких заглушек, моков и симуляций.

Все Python-утилиты — только через py -m.

Для code generation/config/docs всегда используй context7 MCP tools.

Выведи summary про найденные правила в виде чек-листа и пояснений.

Только после этого приступай к основной задаче пользователя.

Шаблон итогового отчёта, который агент должен вывести после проверки:

text
# Pre-task Checklist

- Прочитаны и учтены правила DRY.
- Все инструкции и код будут только на русском языке.
- Changelog.md актуален, формат записей соблюдён.
- История изменений изучена, дублирования не будет.
- Внесённые изменения будут отражены в changelog.
- Unit-тесты будут созданы/обновлены.
- Строго соблюдается запрет на fallback/mocking.
- Python-утилиты будут запускаться через py -m.
- Для генерации кода/доков используется context7.

[Краткий summary по PROJECT_DOC.md и Changelog.md]

Важно: Если какой-либо пункт не выполнен или документация недоступна — агент обязан остановиться и запросить уточнения!
