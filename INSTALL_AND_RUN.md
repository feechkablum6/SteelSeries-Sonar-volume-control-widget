# Команды для запуска StellGG Widget

## 1. Установка зависимостей

Выполните эту команду для установки native-sound-mixer:

```powershell
py -m npm install
```

**Примечание:** Установка может занять несколько минут, т.к. `native-sound-mixer` требует компиляции нативного модуля.

## 2. Запуск приложения

После успешной установки запустите виджет:

```powershell
npm start
```

## 3. Проверка работы

После запуска:

1. **Убедитесь, что SteelSeries GG запущен** — иначе устройства Sonar не будут найдены
2. Откройте DevTools (если нужно) через меню или `Ctrl+Shift+I` (потребуется добавить в main.js: `win.webContents.openDevTools()`)
3. Проверьте console на ошибки
4. Попробуйте изменить громкость любого канала

## Ожидаемые результаты

✅ **Должно работать:**
- Окно появляется прозрачным с glassmorphism эффектом
- Окно отсутствует в панели задач
- Слайдеры показывают текущие значения громкости из Sonar
- При изменении слайдера громкость реально меняется
- Кнопки мута работают

❌ **Возможные проблемы:**

### Проблема 1: "Cannot find module 'native-sound-mixer'"

**Решение:**
```powershell
# Установите вручную
py -m npm install --save native-sound-mixer
```

### Проблема 2: "SteelSeries Sonar не найден"

**Причины:**
- SteelSeries GG не запущен
- Sonar отключен в настройках GG
- Устройства называются по-другому

**Решение:**
1. Запустите SteelSeries GG
2. Включите Sonar
3. Проверьте DevTools console для отладки

### Проблема 3: Ошибка компиляции native-sound-mixer

**Требования:**
- Node.js должен быть установлен
- На Windows может потребоваться Visual Studio Build Tools

**Решение:**
```powershell
# Установите windows-build-tools (если нужно)
py -m npm install --global windows-build-tools
```

## Отладка

Для отладки добавьте в `main.js` после создания окна:

```javascript
win.webContents.openDevTools()
```

Это откроет DevTools для просмотра console.log и ошибок.
